# Training logs — Financial Triage, Qwen2.5-7B-Instruct, SFT → GRPO

Exactly what a reviewer asked for: the **env**, the **training scripts**, the **training logs**.

| Artifact | What it is |
|---|---|
| [`training_run.json`](training_run.json) | Every hyperparameter + every number the committed notebook preserves. Machine-readable. |
| [`training_loss.png`](training_loss.png) | SFT per-step training loss (60 optimizer steps, bs=8 effective, lr=2e-4 → linear decay). Generated by the notebook's plot cell from `trainer.state.log_history`. |

## The three pieces the judge asked for

**1. Environment** — <https://huggingface.co/spaces/indra-dhanush/financial-triage-env>
- Full OpenEnv HTTP contract: `POST /reset`, `POST /step`, `GET /state`, `GET /schema`, `GET /metadata`, `GET /health`, `GET /mcp`.
- Live walkthrough UI at [`/demo/`](https://indra-dhanush-financial-triage-env.hf.space/demo/).
- 15-check compliance script: `python -m scripts.check_openenv --base <url>`.
- Mirrored on GitHub: <https://github.com/indra-2007/financial-triage-env>.

**2. Training scripts** (runnable standalone on any GPU — also embedded in the notebook)
- [`scripts/train_sft.py`](../scripts/train_sft.py) — SFT warmup on `sft_dataset.jsonl` (180 rows of heuristic rollouts), 60 optimizer steps, LoRA-r16 on Qwen2.5-7B-4bit.
- [`scripts/train_grpo.py`](../scripts/train_grpo.py) — GRPO on top of the SFT checkpoint. Reward replays the expert prefix, strictly parses the model's action, runs one `env.step`, and optimizes the scaled dense return — no heuristic fallback.
- [`financial_triage_training.ipynb`](../financial_triage_training.ipynb) — Colab one-click version; same code, same config.
- Reward function invariants asserted by [`tests/test_reward_properties.py`](../tests/test_reward_properties.py) (pytest).

**3. Training logs**
- `training_loss.png` — per-step SFT loss across 60 steps.
- `training_run.json` — every hyperparameter plus the three-point comparison surfaced at eval time:
  | Task | Heuristic (n=5) | SFT (n=5) |
  |---|---:|---:|
  | easy   | 0.999  | 0.933 |
  | medium | 0.694  | 0.596 |
  | hard   | 0.427  | 0.405 |
- `before_after_scores.png` — heuristic vs SFT vs GRPO, n=5 seeds, all three difficulties.

## What is NOT in this folder (honest accounting)

- **Per-step SFT loss JSON** (`outputs/sft/trainer_state.json`) is **not** committed. The PNG preserves the curve, but the raw log history was not saved to disk during the Colab run. Re-run `python -m scripts.train_sft` on any GPU and it will write the full `trainer_state.json` next to the checkpoints.
- **GRPO per-step log** is similarly not committed. Re-run `python -m scripts.train_grpo` after SFT.
- **A public W&B URL**. Both training scripts read `WANDB_API_KEY` from env and set `report_to='wandb'` when it is present — so a judge re-running the scripts with their own key gets the full live log. I did not embed an API key or my W&B URL in the repo.

These gaps exist because I only had one Colab T4 session to produce the artifacts that are committed; re-running to capture the missing JSON requires GPU time I can burn on request.

## Config at a glance

**SFT** · 180-row heuristic replay · batch 1 × grad_accum 8 × 60 steps · lr 2e-4 → linear decay · LoRA r=16 α=16 on all Qwen attention + MLP projections · seed 3407 · bf16/fp16 mixed · adamw_8bit · packing on · max_seq_length 1024.

**GRPO** · 180 prompt rows (3 tasks × 3 seeds × 20 days) · batch 2 × grad_accum 4 × 1 epoch · lr 5e-5 · 4 generations per prompt · max_completion_length 64 · reward ∈ [-1, 1] scaled from `_last_breakdown['total']/30` · malformed action → reward -1 · optional W&B mirroring.

## One-command reproduction (GPU)

```bash
git clone https://huggingface.co/spaces/indra-dhanush/financial-triage-env
cd financial-triage-env
pip install -e .
pip install "unsloth[colab-new] @ git+https://github.com/unslothai/unsloth.git" \
            "trl==0.11.*" "transformers>=4.44" "datasets" "accelerate" "bitsandbytes"

export WANDB_API_KEY=<optional>

python -m scripts.train_sft           # writes outputs/sft/  (trainer_state.json + LoRA)
python -m scripts.train_grpo          # writes outputs/grpo/ (trainer_state.json + LoRA)
```

Or open `financial_triage_training.ipynb` in Colab and pick **Runtime → Run all**. Tesla T4 is sufficient.
